## What is DREAM?

DREAM (Decentralized, Rights-preserving, Encrypted Application Model) is an open application model built on the Verus Protocol. It was introduced by Mike Toutonghi at Paris Blockchain Week 2026.

### Core Principles

- **Self-sovereign identity** — Rent-free, permanent, worldwide-resolvable IDs controlled by their owners forever
- **Privacy by default** — All user data encrypted with keys only the user holds
- **No data exploitation** — No selling of user data, no surveillance capitalism
- **Fair payments** — Sub-cent transaction fees, no MEV extraction, basket currencies with equitable pricing
- **Agent-native** — Built for both human and AI agent interaction

### The Problem DREAM Solves

Legacy applications create a **power asymmetry**: a middle tier of servers and companies sits between users and their data, collecting, selling, and controlling access to personal information.

DREAM redefines the data tier to include a **blockchain as a public, permanent database with encryption under full user control**, removing the middle tier's ability to exploit user data.

### Key Capabilities

| Feature | Description |
|---------|-------------|
| **VerusID** | Self-sovereign identity with indexed key/value database and built-in encryption |
| **Post-quantum password manager** | Encrypted credential storage, 2FA by default |
| **Basket currencies** | Conversion without MEV extraction — everyone gets the same price |
| **Selective disclosure** | Prove only what is needed (e.g., "over 18") without revealing full identity |
| **Cross-device deeplink + QR** | Login from any device without passwords |
| **E2E encrypted messaging** | Built into DREAM apps |
